# How to Report a Bug

Use the GitHub issue templates when something in Aira is broken, inconsistent, or unexpectedly difficult to use.

## Open the Bug Form

- Go to [Issues](../../issues)
- Click [New issue](../../issues/new/choose)
- Select **Bug Report**

## What Makes a Good Bug Report

Include the exact problem, not just the result.

Useful details:

- what you were trying to do
- what you expected to happen
- what actually happened
- whether it happens every time or only sometimes
- your Aira version
- your macOS version
- your Mac model
- whether you were using the built-in display, an external display, or both
- whether microphone, speech recognition, or accessibility permissions were granted

## Especially Helpful for Aira

Because Aira depends on macOS-specific behavior, these details matter a lot:

- notch Mac or non-notch Mac
- Apple Silicon or Intel
- menu bar usage
- overlay mode used: notch, pill, or both
- whether voice sync was enabled
- whether the bug only appears after relaunch

## Screenshots and Recordings

If the bug is visual, include:

- a screenshot
- a short screen recording
- or both

If the issue is about overlays not appearing in the right place, showing your display setup helps.

## Before Submitting

- check if the issue already exists
- use one issue per bug
- keep the title specific

Good title:

`Notch overlay appears on external monitor after reconnecting display`

Bad title:

`Overlay broken`
