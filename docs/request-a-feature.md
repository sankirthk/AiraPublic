# How to Request a Feature

Use the GitHub feature request template when you want Aira to support a new workflow, control, or quality-of-life improvement.

## Open the Feature Form

- Go to [Issues](../../issues)
- Click [New issue](../../issues/new/choose)
- Select **Feature Request**

## What Makes a Good Feature Request

Start with the problem, not the solution.

Useful structure:

1. what you are trying to do
2. what currently gets in the way
3. what you want Aira to support
4. why it would be valuable

## Good Examples

- faster collection-based session launching
- better external-display behavior for fullscreen overlays
- improved speaking aids for accessibility
- more flexible session controls for live presentations

## Less Helpful Requests

These are harder to act on:

- `make it better`
- `improve UI`
- `add more settings`

Be specific about the workflow.

## Scope Guidance

Feature requests are most useful when they describe:

- one clear user need
- one workflow
- one missing behavior

If you have several unrelated ideas, open separate issues.
